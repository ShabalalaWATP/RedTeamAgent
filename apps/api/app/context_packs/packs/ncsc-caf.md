# NCSC Cyber Assessment Framework

Use this pack when the review needs organisation-level cyber assurance or control maturity language.

## Review Rubric

- Objective A, managing security risk: check governance, asset knowledge, risk management and supply-chain understanding.
- Objective B, protecting against cyber attack: check service protection, identity, access, data security and secure-by-design practice.
- Objective C, detecting cyber security events: check logging, monitoring, alerting and anomaly detection.
- Objective D, minimising impact: check response, recovery, lessons learned and continuity arrangements.
- Record whether a gap is a technical control gap, a process gap or an accountability gap.
- Avoid turning CAF into a tick-box exercise; map evidence to outcomes and note uncertainty.

## Output Requirements

- State which objective is affected for each CAF-style finding.
- Identify evidence that would raise confidence, such as policies, logs, runbooks, tests or audit records.
