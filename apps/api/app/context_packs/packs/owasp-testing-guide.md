# OWASP Web Security Testing Guide

Use this pack for web security testing methodology and validation planning.

## Review Rubric

- Confirm scope, rules of engagement and test data before testing.
- Cover information gathering, configuration, identity, session, authorisation, input validation, error handling, cryptography, business logic, client-side security and API behaviour.
- Treat unauthorised, destructive or high-volume testing as out of scope unless explicitly approved.
- Prefer reproducible, minimal test cases with clear expected and observed behaviour.
- Record evidence locators, HTTP request context and affected roles without storing secrets.
- Calibrate severity using exploitability, impact, required privileges, user interaction and compensating controls.

## Output Requirements

- Distinguish validated findings from hypotheses.
- Include safe reproduction steps only for owned or authorised targets.
