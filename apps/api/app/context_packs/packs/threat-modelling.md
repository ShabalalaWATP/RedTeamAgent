# Threat Modelling Review Rubric

Use this pack when the review needs structured adversarial thinking.

## Review Rubric

- Identify assets, actors, trust boundaries, data flows and privileged operations.
- Consider spoofing, tampering, repudiation, information disclosure, denial of service and privilege escalation.
- Consider abuse cases: account takeover, tenant escape, malicious file upload, prompt injection, provider misuse and admin abuse.
- Map each threat to preventive, detective and recovery controls.
- Mark unresolved threats as assumptions or evidence gaps, not as accepted safety.

## Output Requirements

- Each threat should name attacker capability, target asset, expected impact and a control.
- Avoid speculative threats that cannot change a decision.
