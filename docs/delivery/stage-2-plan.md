# Stage 2 Implementation Plan

## Success Criteria

Stage 2 is complete only when Stage 1 still passes and the Stage 2 Definition of Done in `docs/codex-three-stage-goals.md` passes or incomplete items are listed as release-blocking.

## Work Items

1. Update architecture, data model, workflow, threat model and ADRs for Stage 2.
2. Add rich source ingestion for presentations, spreadsheets, images, audio, video, websites, code archives and public repositories.
3. Add hardened fetch, archive-safety and source-redaction policies with tests.
4. Add external research settings, deterministic search-provider contracts and reproducible external source records.
5. Expand provider registry, capability contracts, probes and model-diversity routing.
6. Expand the specialist registry and router to cover all Stage 2 agents.
7. Add advanced report sections, action tracking, report comparison and PDF export.
8. Add deterministic evaluation fixtures, metrics and adversarial cases.
9. Add frontend controls for rich uploads, research settings, report comparison, risk matrix and action tracking.
10. Run Stage 1 and Stage 2 quality, security, accessibility, visual, performance, Docker and documentation gates.

## Out Of Scope

Stage 3 features remain out of scope until Stage 2 passes.
