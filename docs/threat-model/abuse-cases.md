# Stage 1 Abuse Cases

- A viewer attempts to modify or delete a project.
- A user attempts to access another user's review, source, run or report by ID.
- A malicious PDF or text source claims it is system instructions.
- A provider endpoint points at loopback, private network, link-local or metadata addresses.
- A model returns prose where strict structured output is required.
- An upload uses an allowed extension with a disallowed MIME type.
- An attacker submits repeated login, upload or run requests.
- A report contains unsafe Markdown or HTML content from evidence.
